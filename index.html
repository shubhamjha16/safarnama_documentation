Project Overview

The Indian Railways Platform Information Chatbot is an AI-powered WhatsApp assistant designed to reduce passenger confusion at busy railway stations by providing instant access to critical information, including:

  • Real-time platform numbers
  • Train arrival and departure times
  • Delay status updates
  • Station-specific train schedules

Did you know? Indian Railways operates over 22,000 trains daily, serving more than 8 billion passengers annually across 7,325 stations.

System Architecture

Architecture Diagram

Core Components

  • 1
    Chatbot Logic: Processes user queries, extracts intent, and orchestrates responses.
  • 2
    NTES API Interface: Connects to National Train Enquiry System to fetch real-time train data.
  • 3
    Subscription Database: Stores user subscriptions and preferences.
  • 4
    WhatsApp Integration: Handles messaging through the WhatsApp Business API.
  • 5
    Update Monitor: Tracks changes in train status to trigger notifications.

Data Flow

  1. User sends a query via WhatsApp
  2. WhatsApp Business API forwards the message to the chatbot
  3. Chatbot processes the query and determines the required action
  4. For information requests, data is fetched from the NTES API
  5. For subscriptions, user details are stored in the database
  6. Response is formatted and sent back to the user via WhatsApp
  7. For subscribed trains, any status changes trigger notifications

Core Code Components

1. NTES API Interface

 class NTESInterface:
     def __init__(self, api_key):
         self.api_key = api_key
         self.base_url = "https://api.ntes.indian-railways.gov.in/v1/"
         
     def get_train_info(self, train_number):
         """Fetch train details including platform, schedule, delays"""
         endpoint = f"trains/{train_number}/status"
         response = self._make_request(endpoint)
         return response
         
     def get_station_schedule(self, station_code):
         """Fetch all trains at a specific station"""
         endpoint = f"stations/{station_code}/schedule"
         response = self._make_request(endpoint)
         return response
         
     def _make_request(self, endpoint):
         headers = {"Authorization": f"Bearer {self.api_key}"}
         url = self.base_url + endpoint
         response = requests.get(url, headers=headers)
         return response.json() if response.status_code == 200 else None

2. Subscription Manager

 class SubscriptionManager:
     def __init__(self, db_connection):
         self.db = db_connection
         
     def subscribe(self, phone_number, train_number):
         """Add a new subscription"""
         query = """
         INSERT INTO subscriptions (phone_number, train_number, created_at) 
         VALUES (?, ?, datetime('now'))
         """
         self.db.execute(query, (phone_number, train_number))
         self.db.commit()
         
     def unsubscribe(self, phone_number, train_number):
         """Remove a subscription"""
         query = """
         DELETE FROM subscriptions 
         WHERE phone_number = ? AND train_number = ?
         """
         self.db.execute(query, (phone_number, train_number))
         self.db.commit()
         
     def get_subscribers(self, train_number):
         """Get all subscribers for a specific train"""
         query = """
         SELECT phone_number FROM subscriptions 
         WHERE train_number = ?
         """
         result = self.db.execute(query, (train_number,))
         return [row[0] for row in result.fetchall()]

Key Technical Challenges

  • Real-time Updates: Implementing an efficient polling mechanism to detect train status changes without overwhelming the NTES API.
  • Message Handling: Creating a robust NLP system to understand various ways users might phrase their queries.
  • Scalability: Designing the system to handle thousands of concurrent users during peak travel seasons.
  • Reliability: Ensuring high availability even when the NTES API experiences downtime by implementing caching and fallback mechanisms.

Setup Instructions

Prerequisites

  • Python 3.9+ installed
  • WhatsApp Business API account
  • NTES API access credentials
  • Docker (optional, for containerized deployment)

Installation Steps

1. Clone the repository

 git clone https://github.com/yourusername/railway-chatbot.git
 cd railway-chatbot

2. Set up a virtual environment

 python -m venv venv
 source venv/bin/activate  # On Windows: venv\Scripts\activate

3. Install dependencies

 pip install -r requirements.txt

4. Configure environment variables

 # Create a .env file with the following values
 WHATSAPP_API_TOKEN=your_whatsapp_token
 NTES_API_KEY=your_ntes_api_key
 DATABASE_URL=sqlite:///chatbot.db  # For development

5. Initialize the database

 python init_db.py

6. Run the application

 python app.py

7. Configure WhatsApp webhook

In your WhatsApp Business API dashboard, set up a webhook that points to:

 https://your-server-domain.com/webhook
